## What's New in v3.1.0

### 🎆 System Control Framework (NEW)

A new unified framework for controlling system-level features with consistent APIs across all platforms.

#### Volume Control
- **GetVolume:** Read current volume level (0.0-1.0 range)
- **SetVolume:** Set volume to specific level with validation
- **MuteVolume:** Mute/unmute system audio

**Platforms:**
- Linux: PulseAudio (pactl) + ALSA fallback (amixer)
- macOS: osascript (system volume)
- Windows: nircmd + PowerShell fallback

#### Brightness Control
- **GetBrightness:** Read current display brightness
- **SetBrightness:** Set brightness with 0.0-1.0 range
- **Helper methods:** Increase/decrease brightness by 10%

**Platforms:**
- Linux: brightnessctl + xbacklight fallback
- macOS: osascript (System Events)
- Windows: PowerShell/WMI with nircmd fallback

#### Media Control
- **MediaPlayPause:** Toggle play/pause on current player
- **MediaNext:** Skip to next track
- **MediaPrevious:** Go to previous track
- **MediaStop:** Stop playback

**Platforms:**
- Linux: playerctl (MPRIS) + XF86Audio keys
- macOS: osascript (Apple Music) + key codes
- Windows: nircmd + virtual media keys

### 🔄 Hybrid Execution Model

All system controls implement a **hybrid execution strategy**:
1. **Primary:** Command-line tools (reliable when available)
2. **Fallback:** Keyboard simulation (always works)

This ensures graceful degradation and maximum compatibility.

✅ **Linux:** Full support (Debian, Ubuntu, Fedora, Arch, etc.)

## Migration Guide

### From v3.0 to v3.1

**No migration needed.** Simply replace the binary:

```bash
# Backup current version
cp /usr/local/bin/axon-desktop-agent \
   /usr/local/bin/axon-desktop-agent.3.0

# Deploy new version
cp target/release/axon-desktop-agent \
   /usr/local/bin/axon-desktop-agent

# Restart service
pkill -f axon-desktop-agent
/usr/local/bin/axon-desktop-agent &
```

All v3.0 features continue to work. New v3.1 features (9 new RPCs) are immediately available.

## Known Limitations & Future Work

### Current Limitations
1. **Windows Brightness:** PowerShell/WMI implementation is stub (may not work on all drivers)
2. **macOS Media:** Limited to Apple Music app (other players via fallback only)
3. **TLS/Security:** gRPC runs unencrypted (suitable for internal networks)
4. **Authentication:** No built-in auth (firewall rules recommended)

### Future Enhancements (v3.2+)

**Planned Features:**
- TLS/SSL encryption for gRPC
- Token-based authentication
- Rate limiting per client
- Batch operations (combine 3+ operations)
- Tool caching (5-10% performance improvement)
- Output caching (10-20% improvement for polling clients)
- Audit logging (persistent operation logs)
- Fine-grained permissions model

**Optimization Opportunities:**
- Implement tool availability caching
- Add command output caching (TTL-based)
- Support batch system control operations
- Keyboard-first execution option

## Installation & Deployment

### System Requirements

**Linux:**
- PulseAudio or ALSA (for volume)
